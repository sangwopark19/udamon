**Part A: public.users 테이블 (per D-01)**

```sql
CREATE TABLE public.users (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  email TEXT,
  nickname TEXT NOT NULL DEFAULT '',
  avatar_url TEXT,
  bio TEXT DEFAULT '',
  role TEXT NOT NULL DEFAULT 'user' CHECK (role IN ('user', 'admin')),
  my_team_id UUID REFERENCES public.teams(id),
  push_token TEXT,
  blocked_users UUID[] DEFAULT '{}',
  nickname_changed_at TIMESTAMPTZ,
  is_photographer BOOLEAN DEFAULT FALSE,
  is_deleted BOOLEAN DEFAULT FALSE,
  deleted_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

칼럼 설명:
- `id`: auth.users(id) FK, ON DELETE CASCADE
- `email`: OAuth에서 가져온 이메일 (nullable -- 일부 OAuth 미제공)
- `nickname`: 기본 빈 문자열, Phase 2에서 온보딩 시 설정
- `role`: 'user' 또는 'admin' (per D-10 -- JWT claims 불사용)
- `my_team_id`: teams FK (nullable -- 미선택 가능)
- `push_token`: FCM 토큰 (Phase 6용)
- `blocked_users`: 차단 사용자 UUID 배열 (Phase 2 AUTH-10용)
- `nickname_changed_at`: 닉네임 변경 일시 (Phase 2 30일 제한용)
- `is_photographer`: 포토그래퍼 여부 (Phase 4용)
- `is_deleted`, `deleted_at`: soft delete (per D-03)

**Part C: handle_new_user 트리거 함수 (per D-02)**

```sql
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER
LANGUAGE plpgsql
SECURITY DEFINER SET search_path = ''
AS $$
BEGIN
  INSERT INTO public.users (
    id,
    email,
    nickname,
    avatar_url,
    role
  ) VALUES (
    new.id,
    new.email,
    COALESCE(new.raw_user_meta_data ->> 'username', 'user_' || LEFT(new.id::text, 8)),
    new.raw_user_meta_data ->> 'avatar_url',
    'user'
  );
  RETURN new;
END;
$$;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE PROCEDURE public.handle_new_user();
```

주의: `SECURITY DEFINER SET search_path = ''`는 필수 (Supabase Security Advisor). 함수 본문에서 `public.users`로 스키마 명시.
