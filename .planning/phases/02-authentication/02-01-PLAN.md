<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Supabase Auth | OAuth 토큰, 이메일/비밀번호가 네트워크를 통해 전송 |
| OAuth Provider -> App | OAuth redirect URL을 통해 code/token이 앱으로 전달 |
| App -> AsyncStorage | 세션 토큰이 디바이스 로컬 저장소에 저장 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | OAuth redirect | mitigate | PKCE가 Supabase에 기본 활성화, skipBrowserRedirect: true로 서버 사이드 redirect 방지, 등록된 redirect URL만 허용 |
| T-02-02 | Information Disclosure | AsyncStorage token | accept | v1은 AsyncStorage 사용 (Supabase 공식 권장), autoRefreshToken으로 토큰 수명 최소화. v2에서 SecureStore 검토 |
| T-02-03 | Elevation of Privilege | service_role key | mitigate | service_role 키는 Edge Function에서만 사용, 클라이언트 코드에 절대 포함하지 않음. anon key만 클라이언트에 노출 |
| T-02-04 | Tampering | signInWithPassword | mitigate | Supabase 서버 사이드에서 bcrypt 해싱 처리, 클라이언트에서 비밀번호 직접 처리 안 함 |
| T-02-05 | Denial of Service | login attempts | accept | v1에서는 Supabase의 기본 rate limiting에 의존. 추가 throttling은 v2 검토 |
| T-02-06 | Spoofing | session fixation | mitigate | onAuthStateChange에서 SIGNED_IN 이벤트 시에만 프로필 로드, TOKEN_REFRESHED에서는 세션만 교체 |
</threat_model>
