# Phase 4 — UI Design Contract

> Visual and interaction contract for Photographer phase (React Native Expo mobile app).
> Generated by gsd-ui-researcher from REQUIREMENTS.md (PHOT-01~08) + 04-CONTEXT.md (D-01~D-23) + existing theme tokens (`app/src/styles/theme.ts`, `app/src/constants/colors.ts`).
> Do NOT re-invent spacing/typography/color tokens — they are locked in the theme. This contract binds component variants, state copy, and new component specs only.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | **none** (React Native — shadcn gate not applicable) |
| Preset | not applicable |
| Component library | React Native built-ins + custom (`app/src/components/`) |
| Icon library | `@expo/vector-icons` — **Ionicons** family only |
| Font | System default (iOS San Francisco / Android Roboto) |
| Token source | `app/src/styles/theme.ts` + `app/src/constants/colors.ts` |
| i18n | `react-i18next` — all user-facing strings via `useTranslation()` keys in `app/src/i18n/locales/ko.ts` |

**Pattern rules (locked by existing codebase):**
- All color, spacing, typography, radius values **MUST** come from `theme.ts` / `colors.ts`. No raw hex, no literal px outside the token files.
- Inline `StyleSheet.create` per screen/component (established pattern).
- `activeOpacity={0.7}` on every `TouchableOpacity`.
- Safe area handled via `useSafeAreaInsets()` inline styles.

---

## Spacing Scale

Locked tokens from `app/src/styles/theme.ts` (4-point grid). Do not introduce new values:

| Token | Value | Usage in Phase 4 |
|-------|-------|------------------|
| `spacing.xs` | 4px | Icon↔text gap inside badge, stack dot spacing |
| `spacing.sm` | 8px | Tight inline spacing (chip padding, video badge inset) |
| `spacing.md` | 12px | Form field vertical gaps, card internal padding rows |
| `spacing.lg` | 16px | Screen horizontal padding, card outer padding, grid gutters |
| `spacing.xl` | 24px | Section breaks between grouped blocks |
| `spacing.xxl` | 32px | Major empty-state/pending-state vertical rhythm |

Additional layout constants from `theme.ts` (reuse, do not re-declare):
- `layout.headerHeight = 56` (screen top bar height)
- `layout.tabBarHeight = 70` (bottom tab bar)

**Phase 4 exceptions:** none. All new components must consume these tokens.

**Touch targets:**
- Minimum 44×44 for all pressable elements (platform A11y).
- VideoPlayer play/pause overlay: 56×56 (icon button centered on video).
- Grid video play-icon overlay (gallery): 32×32 centered (decorative — container is fully pressable).

## Typography

Locked tokens from `theme.ts` `fontSize` + `fontWeight`. **No new sizes/weights introduced.** Role mapping for Phase 4 surfaces:

| Role | Token (size) | Token (weight) | Line Height | Used In |
|------|--------------|----------------|-------------|---------|
| Screen title | `fontSize.sectionTitle` = 21px | `fontWeight.heading` = 800 | 28 | `studio_title`, `pending_review_title`, `rejected_title` |
| Section heading / badge tier name | `fontSize.cardName` = 16px | `fontWeight.heading` = 800 | 22 | Grade tier labels, pending/rejected card titles |
| Body / CTA label | `fontSize.body` = 15px | `fontWeight.body` = 600 | 22 | Primary CTA buttons, descriptive copy |
| Meta / helper | `fontSize.meta` = 13px | `fontWeight.body` = 600 | 20 | Hint text ("영업일 기준 2~3일"), form counters |
| Micro | `fontSize.micro` = 12px | `fontWeight.body` = 600 | 16 | Video duration chip, file-size warning |
| Badge label | `fontSize.badge` = 11px | `fontWeight.name` = 700 | 14 | `GradeBadge` tier name (icon+label variant) |

Heading stack (largest → smallest): `display 30` → `sectionTitle 21` → `price 18` → `cardName 16`.
Body stack: `body 15` → `meta 13` → `micro 12` → `badge 11` → `tiny 10` → `micro2 9`.

Phase 4 **only uses** the 6 roles listed above. Avoid `display`, `price`, `tiny`, `micro2` in the new surfaces — they exist for other domains.

## Thumbnail Fallback Rendering

Per D-15 (Phase 4 new posts only):

```
const previewUrl = (post.thumbnail_urls?.[0]) ?? post.images[0];
```

- Immediately after upload, Edge Function fire-and-forget may not have completed → `thumbnail_urls` is `[]` → render original `images[0]`. Sized via `Image` component `resizeMode="cover"` at grid/feed dimensions.
- `ProgressiveImage` component (`app/src/components/common/ProgressiveImage.tsx`) handles blur-up loading; reuse unchanged.
- On subsequent feed refresh, once Edge Function completes, `thumbnail_urls[0]` resolves → smaller payload served transparently.
- No UX difference visible to user — this is the contract's primary goal (CONTEXT §Specifics "사용자는 실패를 느끼지 않음").
