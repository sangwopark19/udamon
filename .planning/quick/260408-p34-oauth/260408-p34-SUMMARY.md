# Quick Task 260408-p34: Android Kakao OAuth callback fix

## What Changed

### app/src/contexts/AuthContext.tsx
- Added `ensureUserProfile(authUser)` helper: tries `fetchUserProfile` first, falls back to upsert from `user_metadata` if null
- Replaced direct `fetchUserProfile` calls with `ensureUserProfile` in 3 locations: `init()`, `onAuthStateChange`, `refreshUser`
- Added diagnostic logs: `[Auth] onAuthStateChange event`, `[Auth] Profile not found, creating`, `[OAuth] setSession success`

## Root Cause

OAuth callback returned valid tokens, `supabase.auth.setSession()` succeeded, but `fetchUserProfile()` returned null because `public.users` had no row for the auth user. The `handle_new_user` DB trigger either wasn't applied or lost a race condition. Without a profile row, `setUser(null)` kept `isAuthenticated = false`, showing the login screen.

## Fix

Client-side fallback: when `fetchUserProfile` returns null, `ensureUserProfile` upserts a minimal profile from auth `user_metadata` (name, email, avatar), then re-fetches to get DB defaults. The upsert uses `onConflict: 'id'` so it's safe even when the trigger succeeds.
