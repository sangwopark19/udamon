# Quick Task 260423-gz3 — 영상 썸네일 네비게이션 실패 수정

## 문제 (Bug)

안드로이드 실기기에서 "이번 주 베스트 컷" 및 유사 영역(홈 트렌딩, AllPosts, FeaturedAll, 포토그래퍼 프로필, 아카이브 등)에서:
- 이미지 썸네일 탭 → 게시글 상세 정상 이동 ✅
- 영상 썸네일 탭 → **반응 없음** ❌

## 근본 원인 (Root Cause)

`app/src/components/common/VideoPlayer.tsx`의 `<VideoView>` (expo-video)가 네이티브 레벨에서 터치 이벤트를 자기 자신이 소비한다. 카드 래퍼는 다음 구조:

```jsx
<TouchableOpacity onPress={() => navigation.navigate('PostDetail', { postId })}>
  {hasVideo ? <VideoPlayer mode="feed" /> : <Image />}
</TouchableOpacity>
```

- **이미지 케이스:** `<Image>`는 기본적으로 touch를 가로채지 않아 부모 `TouchableOpacity`로 이벤트가 버블 → `onPress` 실행.
- **영상 케이스:** `<VideoView>` (Android `PlayerView`)는 `feed`/`studio` 모드에서도 터치를 소비 → 부모 `onPress`가 호출되지 않음.

`mode="detail"` 에서는 `nativeControls`가 활성화되어 VideoView가 터치를 받는 것이 올바르므로 이 경로는 건드리지 않는다.

## 영향 범위 (Affected Surfaces)

## 수정 전략 (Fix Strategy)

`VideoPlayer.tsx`의 최상위 래퍼 `<View>`에 `pointerEvents`를 모드별로 설정:
- `mode === 'detail'` → `pointerEvents="auto"` (기존 동작 유지, nativeControls 가 터치 수용)
- `mode === 'feed' | 'studio'` → `pointerEvents="none"` (자기 자신과 모든 자식이 터치 비활성 → 부모 Pressable 로 이벤트 전달)

React Native 0.81 에서는 `pointerEvents`를 prop 으로도 style 로도 지정 가능. 호환성/안전 위해 **prop 형태**로 지정 (VideoView 상위 `<View>` 에 적용).
