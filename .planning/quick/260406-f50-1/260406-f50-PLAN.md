다음 섹션들을 포함해야 한다:

## 1. 사전 준비 (Prerequisites)
- 필수 소프트웨어: Node.js, npm, Supabase CLI (v2.x), Docker Desktop (로컬 개발 시)
- Supabase 프로젝트 생성: https://supabase.com 에서 프로젝트 생성 절차
- Supabase 대시보드에서 Project URL과 anon key, service_role key 확인 위치

## 2. Supabase CLI 프로젝트 링크
- `supabase login` 명령으로 인증
- `supabase link --project-ref <project-ref>` 명령으로 프로젝트 연결
  - project-ref는 Supabase 대시보드 > Project Settings > General에서 확인
- 링크 성공 확인: `supabase status` 명령 결과 예시

## 3. 환경변수 설정
### 3.1 모바일 앱 (app/.env)
- `cp app/.env.example app/.env`
- 필수 변수: EXPO_PUBLIC_SUPABASE_URL, EXPO_PUBLIC_SUPABASE_ANON_KEY
- 값 확인 위치: Supabase Dashboard > Project Settings > API
- 환경변수 미설정 시 에러 메시지 설명 (supabase.ts의 throw Error 동작)

### 3.2 어드민 웹 (admin/.env)
- `cp admin/.env.example admin/.env`
- 필수 변수: VITE_ADMIN_EMAIL, VITE_ADMIN_PASSWORD
- 이 값들은 사용자가 직접 설정하는 어드민 로그인 자격증명
- Phase 5에서 Supabase Auth로 전환 예정이므로 임시 값 사용

### 3.3 Edge Function Secrets (R2 업로드용)
- `supabase secrets set R2_ACCOUNT_ID=xxx R2_ACCESS_KEY_ID=xxx R2_SECRET_ACCESS_KEY=xxx R2_BUCKET_NAME=xxx R2_PUBLIC_URL=xxx`
- 각 값의 확인 위치: Cloudflare Dashboard > R2 > API Tokens, Bucket Settings
- 주의: Phase 1 테스트에서는 Edge Function 배포는 선택 사항 (DB 스키마 검증이 주요 목표)

  - 001~010: 기존 테이블 (teams, players, community, polls, spam_filter, photographer 등)

## 5. Phase 1 Success Criteria 테스트 절차
01-VERIFICATION.md의 human_needed 항목 3개에 대한 구체적 테스트:

### 5.1 테스트 1: auth.users 트리거 동작 확인 (VERIFICATION #1)
- 단계:
  1. Supabase Dashboard > Authentication > Users > "Add user" 클릭
  2. 이메일/비밀번호로 테스트 사용자 생성 (예: test-phase1@example.com / TestPass123!)
  3. Dashboard > Table Editor > public.users 테이블 확인
- 기대 결과: auth.users에 생성된 사용자와 동일한 id로 public.users에 행이 자동 생성됨
  - email: test-phase1@example.com
  - role: 'user' (기본값)
  - nickname: NULL (아직 미설정)
- SQL Editor에서 직접 확인하는 방법:
  ```sql
  SELECT id, email, role, nickname, created_at
  FROM public.users
  ORDER BY created_at DESC
  LIMIT 5;
  ```

### 5.2 테스트 2: RLS 비인증 차단 확인 (VERIFICATION #3)
- 방법 A: Supabase Dashboard > SQL Editor에서:
  ```sql
  -- 현재 세션을 anon role로 전환
  SET ROLE anon;
  SELECT * FROM public.users;
  -- 기대 결과: 빈 결과 (0 rows)
  RESET ROLE;
  ```
- 방법 B: JavaScript로 테스트 (anon key만 사용):
  ```javascript
  import { createClient } from '@supabase/supabase-js';
  const supabase = createClient('YOUR_SUPABASE_URL', 'YOUR_ANON_KEY');
  // 로그인하지 않은 상태에서 조회
  const { data, error } = await supabase.from('users').select('*');
  console.log(data); // 기대: [] (빈 배열)
  console.log(error); // 기대: null (에러 없이 빈 결과)
  ```
- 추가 테이블도 동일하게 테스트: notifications, announcements, inquiries, audit_logs
  ```sql
  SET ROLE anon;
  SELECT count(*) FROM notifications;      -- 0
  SELECT count(*) FROM announcements;       -- 0
  SELECT count(*) FROM audit_logs;          -- 0
  RESET ROLE;
  ```

### 5.3 테스트 3: 전체 테이블 존재 확인 (VERIFICATION #2)
- SQL Editor에서:
  ```sql
  SELECT table_name
  FROM information_schema.tables
  WHERE table_schema = 'public'
  ORDER BY table_name;
  ```
- Phase 1 신규 테이블이 모두 포함되어야 함:
  announcements, audit_logs, cheerleaders, inquiries, notifications,
  photographer_applications, site_settings, users
- 기존 테이블도 존재 확인: teams, players, community_posts, community_comments, photo_posts, photographers 등

### 5.4 테스트 4: 보안 코드 검증 (VERIFICATION #4, #5)

## 6. Edge Function 배포 및 테스트
- 배포: `supabase functions deploy get-upload-url`
- R2 secrets 설정이 완료된 경우 테스트:
  ```bash
  # 1. 먼저 JWT 토큰 획득 (Dashboard에서 생성한 테스트 사용자로)
  curl -X POST 'YOUR_SUPABASE_URL/auth/v1/token?grant_type=password' \
    -H 'apikey: YOUR_ANON_KEY' \
    -H 'Content-Type: application/json' \
    -d '{"email":"test-phase1@example.com","password":"TestPass123!"}'

  # 2. 응답에서 access_token 추출 후 Edge Function 호출
  curl -X POST 'YOUR_SUPABASE_URL/functions/v1/get-upload-url' \
    -H 'Authorization: Bearer YOUR_ACCESS_TOKEN' \
    -H 'Content-Type: application/json' \
    -d '{"prefix":"avatars","contentType":"image/jpeg","count":1}'
  ```
- 기대 결과: `{ "uploads": [{ "uploadUrl": "...", "publicUrl": "...", "key": "..." }] }`
- 인증 없이 호출 시: `{ "error": "Missing or invalid Authorization header" }` (401)

## 7. 앱 실행 및 동작 확인

### 7.1 모바일 앱 (Expo)

### 7.2 어드민 웹 (Vite)
- `cd admin && npm install && npm run dev`
- http://localhost:5173 접속
- admin/.env에 설정한 VITE_ADMIN_EMAIL/VITE_ADMIN_PASSWORD로 로그인
- 주의: 어드민 웹은 아직 mock 데이터 기반 (Phase 5에서 Supabase 연동 예정)

## 8. 트러블슈팅

### 8.1 supabase db push 실패
- "supabase is not linked": `supabase link --project-ref <ref>` 실행
- "relation already exists": 이미 적용된 마이그레이션. `supabase migration list`로 상태 확인
- "permission denied": Supabase Dashboard > Database > Roles에서 postgres role 확인
- Docker 관련 에러: 로컬 개발이 아닌 원격 Supabase 프로젝트에 직접 push하는 경우 Docker 불필요

### 8.2 환경변수 관련
- "Missing required environment variables": app/.env 파일이 없거나 값이 비어있음
- Expo에서 환경변수 변경 후 반영 안 됨: Expo 서버 재시작 필요 (Ctrl+C 후 다시 `npx expo start`)
- VITE 환경변수가 undefined: admin/.env 파일에서 VITE_ 접두사 확인

### 8.3 RLS 관련
- "new row violates row-level security policy": 인증되지 않은 상태에서 INSERT 시도. Supabase Auth 세션이 필요함
- 테이블 조회 결과가 항상 빈 배열: 정상 동작 (인증 세션 없이 조회하면 RLS가 차단)

### 8.4 Edge Function 관련
- "Failed to deploy function": Supabase CLI 버전 확인 (`supabase --version`, 2.x 이상 필요)
- "Internal Server Error" 500: R2 secrets 미설정. `supabase secrets list`로 확인
- CORS 에러 (브라우저): ALLOWED_ORIGINS에 올바른 도메인이 설정되어야 함 (현재 udamonfan.com)
