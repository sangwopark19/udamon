## Deviations from Plan

### Auto-Synced Block Comment in Task 3

**[Rule 2 — Critical correctness/accuracy]** When changing the active regex on line 740 from `/[%,]/g` to `/[%,()]/g`, the explanatory block comment at lines 723-728 still referred to the old regex. The comment was a stale source of truth and would confuse future maintainers. Synced the comment to:

```ts
// WHY sanitize with replace(/[%,()]/g, '')?
// PostgREST `.or('title.ilike.%q%,content.ilike.%q%')` concatenates the
// query string into a PostgREST filter expression — it is NOT parameterized.
// Stripping `%`, `,`, and parens prevents a malicious query from injecting
// extra filter clauses or breaking PostgREST's `.or(...)` parenthesis grouping.
// Length cap (50) limits DoS surface.
```

This adds 5 deletions and 6 insertions on top of the 1-line active-code change but keeps comment and code in sync. Included in the same task-3 commit (`238dcc3`).
