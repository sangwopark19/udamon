## 1. 서비스 개요

### 1.1 기본 정보

| 항목 | 내용 |
|------|------|
| 서비스명 | dugout (더그아웃) |
| 피치 (영문) | "Where every game moment reaches the fans" |
| 피치 (한국어) | "당신이 담은 경기의 순간이 팬에게 닿는 곳" |
| 플랫폼 | iOS/Android 앱 + 웹 (dugoutfan.com) |
| 법인 | 헤이디 (한국) |
| 대상 | KBO 팬 + 팬 포토그래퍼 |
| 디자인 | **라이트 모드 기본** |
| 최소 지원 | **iOS 15+ / Android 9+ (API 28)** |
| 디바이스 | **폰만** (태블릿은 v2) |

### 1.2 서비스 정의

KBO 팬을 위한 **커뮤니티 + 포토그래퍼 플랫폼**.
- **커뮤니티**: 구단별 게시판, 팬 토론, 투표
- **포토그래퍼**: 경기장에서 찍은 선수 사진, 경기 순간, 직캠 공유

> **v1 런칭 방향**: 커뮤니티를 핵심으로 먼저 런칭. 포토그래퍼 업로드는 기본 기능만 포함. 후원/정산은 UI만 유지하고 비활성화 상태로 v2에서 오픈.

### 1.3 법적 포지션

dugout은 UGC(User Generated Content) 호스팅 플랫폼입니다.
- 콘텐츠를 직접 제작하거나 소유하지 않음
- 콘텐츠의 저작권/초상권 책임은 업로드한 포토그래퍼에게 있음
- 구단/KBO와 어떠한 라이선스/초상권 계약도 없음
- 관리자가 등록하는 것은 텍스트(구단명, 선수명)만 — 이미지 0
- 권리 침해 신고 시 즉시 대응 (Notice & Takedown)

## 3. 사용자 구조

### 3.1 사용자 유형

| 유형 | 설명 | 기능 |
|------|------|------|
| 일반 사용자 (팬) | 야구 팬 | 콘텐츠 조회, 검색, 팔로우, 북마크, **커뮤니티 글/댓글/투표** |
| 포토그래퍼 (홈마) | 경기장 팬 포토그래퍼 | 위 + 업로드 + **커뮤니티 글 작성 가능**, ~~후원 수령, 정산~~ _(v2)_ |
| 관리자 | 운영팀 | 전체 관리 |

### 3.2 계정

- 가입: **카카오 / 네이버 / 구글 / 애플 / 이메일**
- 최소 연령: 만 14세 이상
- 닉네임 고유
- **닉네임 변경: 30일에 1회**
- **커뮤니티에서는 닉네임만 표시 (익명)**
- **프로필 사진: 설정 가능**
- **비밀번호 찾기 (이메일 가입자): 임시 비밀번호 발급 후 변경**
- **동시 로그인: 제한 없음**

### 3.3 포토그래퍼 전환

- **누구나 자유 전환** (심사 없음)
- **전환 후 복귀 불가** (포토그래퍼 유지)
- 포토그래퍼도 커뮤니티 글/댓글 작성 가능

### 3.4 탈퇴

- 탈퇴 시 작성자명 **"탈퇴한 사용자"** 로 표시
- 글/댓글은 유지 (삭제되지 않음)

## 4. 온보딩

### 4.1 첫 실행 플로우

```
앱 첫 실행
→ 푸시 알림 권한 요청
→ 간단 가이드 2~3장 (건너뛰기 가능)
→ 로그인/회원가입 (비로그인으로 둘러보기 가능)
→ 마이팀 설정 (선택사항, 건너뛰기 가능)
→ 홈 피드 진입
```

### 4.2 마이팀 설정

- **선택사항** (없이도 진행 가능)
- 설정 시 홈 피드 + 커뮤니티 기본 탭이 해당 구단으로 세팅
- **변경: 언제든 자유 변경 가능**
- 마이 탭 > 설정에서 변경

### 4.3 비로그인 상태

- **홈 피드 열람 가능** (인기 콘텐츠 + 커뮤니티 인기글)
- 좋아요/댓글/팔로우/북마크 등 액션 시 → **로그인 유도 팝업**

## 5. 콘텐츠 구조

### 5.1 카테고리 (포토그래퍼 콘텐츠)

| 카테고리 | 설명 | 예시 |
|----------|------|------|
| 경기 (Game) | 경기 중 촬영 | 타석, 투구, 수비 장면 |
| 선수 (Player) | 선수 개인 사진 | 워밍업, 세리머니, 인터뷰 |
| 직캠 (Fancam) | 영상 콘텐츠 | 홈런 장면, 응원, 하이라이트 |
| 경기장 (Stadium) | 경기장/분위기 | 전경, 야경, 응원석 |
| 비하인드 (Behind) | 훈련/이벤트 | 스프링캠프, 팬미팅, 시구 |

### 5.2 게시물 구조 (포토그래퍼)

**구단 선택 (필수)** → 선수 태그 (선택) → 카테고리 → 경기 이벤트 연결
→ 사진/영상 업로드 (최대 10장, 30MB/장) → 제목 → 해상도 태그 → 게시

### 5.3 포토그래퍼 콘텐츠 댓글

- **댓글 가능** (대댓글 지원, 커뮤니티와 동일 구조)
- 포토그래퍼 본인 콘텐츠의 댓글 삭제 권한 **없음** (신고만 가능)

### 5.4 해상도 태그

8K RAW / 4K 60fps / 4K / FHD / HD (homma와 동일)

### 5.5 타임라인 이벤트

| 이벤트 유형 | 설명 |
|-------------|------|
| regular_season | 정규 시즌 경기 |
| postseason | 포스트 시즌 경기 |
| allstar | 올스타전 |
| spring_camp | 스프링캠프 |
| fan_meeting | 팬미팅/사인회 |

| other | 기타 |

## 6. 커뮤니티 구조 (v1 핵심)

### 6.1 커뮤니티 탭 구조

상단 수평 스크롤 탭:
```
ALL | SSG | 키움 | LG | KT | KIA | NC | 삼성 | 롯데 | 두산 | 한화
```
- **ALL 탭**: 전체 구단 통합 피드 + 인기글/트렌딩
- **구단 탭**: 해당 구단 태그가 걸린 글만 필터링
- **구단 태그 없는 글**: ALL 탭에만 노출

### 6.2 커뮤니티 글 작성

| 항목 | 내용 |
|------|------|
| 작성자 표시 | 닉네임만 (익명) |
| 제목 | 필수, **최대 30자** |
| 본문 | 필수, **최대 1000자** |
| 사진 첨부 | 선택, **최대 10장, 10MB/장** |
| 투표 | 선택 |
| 구단 태그 | **선택사항** — 태그 시 해당 구단 탭에도 노출 |
| 외부 링크 | **허용** (유튜브/뉴스 등) |

### 6.3 글 수정/삭제

- **수정**: 자유 (수정 시 "수정됨" 표시)
- **투표 포함 글**: 수정 불가 → 삭제 후 재작성
- **삭제**: 댓글 포함 전체 삭제

### 6.4 투표 기능

| 항목 | 스펙 |
|------|------|
| 선택지 개수 | 2~6개 |
| 복수 선택 | 글 작성 시 작성자가 선택 (단일/복수) |
| 마감 방식 | 마감 시간 설정 필수 (24시간 / 3일 / 7일) |
| 결과 공개 | **투표 후 즉시 결과 확인** (실시간 반영) |
| 결과 표시 | **바 비율만 표시** (참여자 수 비공개) |
| 재투표 | 불가 (한 번 투표하면 변경 불가) |
| 마감 후 | 결과 고정, "마감됨" 표시, 투표 불가 |

### 6.5 댓글 구조

- **대댓글 지원** (댓글의 대댓글, 1depth까지)
- **댓글 글자 수**: 최대 300자
- 좋아요
- 신고/차단
- **댓글 수정**: 자유
- **댓글 삭제**: "삭제된 댓글입니다" 표시 + 대댓글 유지

### 6.6 커뮤니티 글 목록 뷰

**하이브리드 방식**:
```
[사진 없는 글]
제목                          좋아요 3 | 댓글 12
본문 미리보기 2줄...           5분 전

[사진 있는 글]
제목                    [썸네일]
본문 미리보기 1줄...    [60x60]   좋아요 8 | 댓글 5
```

### 6.7 정렬/필터

- ALL 탭: 최신순 / 인기순 / 트렌딩
- 구단 탭: 최신순 / 인기순

### 6.8 커뮤니티 검색

- **커뮤니티 탭 내 검색바**
- 검색 범위: **제목 + 본문 통합 검색**
- **최근 검색어 저장** (최대 10개, 개별 삭제 가능)

### 6.9 트렌딩 알고리즘

- 기준: 최근 24시간 내 좋아요 + 댓글 수 합산 상위
- **갱신 주기: 1시간마다**
- 활성 투표 진행 중인 글 우선

## 7. 네비게이션 구조 (v1 확정)

### 7.1 하단 탭 (5개)

```
홈 | 탐색 | 아카이브 | 커뮤니티 | 마이
```

| 탭 | 설명 |
|----|------|
| 홈 | 메인 피드 — 포토그래퍼 콘텐츠 + **커뮤니티 인기글 혼합 노출** |
| 탐색 | 구단별/선수별 브라우징 |
| 아카이브 | 검색 + 타임라인 이벤트별 정리 |
| 커뮤니티 | **게시판/팬 토론/투표 (v1 핵심)** |
| 마이 | 프로필/설정/북마크 |

### 7.2 홈 피드 구성 & 혼합 알고리즘

**혼합 비율: 포토그래퍼 콘텐츠 5개당 커뮤니티 인기글 1개**

```
[포토 1] [포토 2] [포토 3] [포토 4] [포토 5]
[🔥 커뮤니티 인기글 카드]
[포토 6] [포토 7] [포토 8] [포토 9] [포토 10]
[🔥 커뮤니티 인기글 카드]
...
```

**커뮤니티 인기글 선정 기준:**
- 최근 24시간 내 좋아요 + 댓글 수 합산 상위
- 활성 투표 진행 중인 글 우선
- 동일 글 중복 노출 방지
- 카드 형태: 제목 + 좋아요/댓글 수 + 구단 태그 + "커뮤니티" 뱃지

**콜드스타트 (팔로우 없을 때):**
- 커뮤니티 인기글 + 포토그래퍼 인기 콘텐츠 혼합 (전체 기준)

**초기 런칭 시 (콘텐츠 부족 상황):**
- 포토그래퍼 콘텐츠가 5개 미만이면 비율 무시, 있는 대로 혼합
- 커뮤니티 글이 부족하면 커뮤니티 카드 건너뛰기

### 7.3 마이 탭 구성

- 프로필 (닉네임, 프로필 사진, 마이팀)
- **내가 쓴 커뮤니티 글**
- **북마크** (포토그래퍼 콘텐츠 + 커뮤니티 글 통합)
- 팔로우/팔로워 목록
- 설정 (닉네임 변경, 마이팀 변경, 알림 설정, 캐시 삭제, 로그아웃, 탈퇴)

### 7.4 피드 UX

- **페이지네이션**: 20개씩 로드
- **pull to refresh**: 지원
- **임시저장**: v1 생략 (v2 검토)

## 17. 법적 방어 체계

### 17.1 핵심 방어 원칙 (5원칙)

```
원칙 1: 콘텐츠에 가격을 매기지 않는다
  → 모든 사진/직캠은 100% 무료 공개
  → 후원은 "콘텐츠"가 아닌 "포토그래퍼(사람)"에게

원칙 2: 관리자가 등록하는 것은 텍스트뿐이다
  → 구단명, 선수명만 텍스트로 등록
  → 관리자가 등록하는 이미지/영상/초상: 0개

원칙 3: 모든 미디어 콘텐츠는 사용자가 올린다

원칙 4: 플랫폼은 콘텐츠를 사전 검증하지 않는다

원칙 5: 구단/KBO와 어떠한 계약도 없다
```

### 17.2 KBO 특수 리스크

**1. 중계권 문제**
- KBO는 방송 중계권을 판매함
- 경기 영상을 직접 캡쳐해서 올리면 중계권 침해 가능
- 하지만: 팬이 경기장에서 직접 촬영한 사진/영상은 중계권과 무관
- 방어: "팬이 경기장에서 직접 촬영한 콘텐츠만 허용"

**2. 구단 초상권**
- 구단이 선수 초상권을 관리
- 하지만: 공개 경기장에서 촬영한 사진은 보도/팬활동 범위
- 방어: UGC 플랫폼 면책 + 무료 콘텐츠 + 상업적 이용 아님

**3. 경기장 촬영 규정**
- 대부분 KBO 구장: 개인 촬영 허용 (플래시/삼각대 제한)
- 상업적 촬영은 별도 허가 필요
- 방어: 팬 개인의 비상업적 촬영물을 호스팅할 뿐

**4. 현실적으로:**
- 트위터/인스타에 이미 수만 장의 야구 팬 사진이 있음
- 구단이 이걸 문제 삼은 적 거의 없음
- 오히려 팬 사진이 구단 홍보에 도움

### 17.3 구단 대응 매뉴얼

```
Level 1: 특정 콘텐츠 삭제 요청
  → DMCA 절차에 따라 48시간 내 처리

Level 2: 포괄 삭제 요청
  → "구체적 URL과 함께 신고해주시면 건건이 처리"
  → 포괄 요청은 법적으로 유효하지 않음

Level 3: 법적 위협
  → 법무 자문 + UGC 면책 주장

Level 4: 구단이 포토그래퍼에게 직접 압력
  → 구단 분산으로 방어 (10개 구단 골고루)
```

### 17.4 중계권 방어 — 업로드 가이드

```
✅ 허용:
  - 경기장에서 본인이 직접 촬영한 사진
  - 경기장에서 본인이 직접 촬영한 영상 (직캠)
  - 훈련장/팬미팅에서 촬영한 콘텐츠

❌ 금지 (업로드 시 경고):
  - TV/OTT 중계 화면 캡쳐
  - 공식 유튜브/SNS에서 가져온 영상
  - 구단 공식 사진 도용
  - 뉴스 기사 사진 도용
```
