## 1. 작업 범위 및 우선순위

> **핵심**: "로그인 연동을 시작으로 백 작업"이라고 하셨는데, 정확한 범위 확인이 필요합니다.

### 1.1 전체 작업 범위

| # | 질문 | 왜 중요한가 |
|---|------|------------|
| 1 | **제가 맡을 작업의 정확한 범위**는 어디까지인가요? 인증만? 커뮤니티 연동까지? 어드민 연동까지? | 작업 시간과 비용 산정의 기본 |
| 2 | 현재 분석 기준 전체 백엔드 작업은 **최소 44시간** 규모입니다. 전체를 맡기시는 건지, 우선순위별로 단계적으로 진행하는 건지? | 일정 계획 |
| 3 | 앱 프론트엔드(React Native) 코드 수정도 제 작업 범위에 포함되나요? (예: Context를 Supabase 연동으로 전환하면 프론트 코드도 바뀜) | 범위 명확화 |
| 4 | 어드민 웹(React+Vite)도 제 작업 범위인가요? | 범위 명확화 |

### 1.2 우선순위 확인

| # | 질문 |
|---|------|
| 5 | 클라이언트 측에서 생각하시는 **v1 런칭 목표 시점**이 있나요? |
| 6 | 런칭 목표에 맞춰 아래 Phase 중 **어디까지가 런칭 필수**인가요? |

```
Phase 1 (~16시간): 인증 정리 + 커뮤니티 Supabase 연동 + 누락 DB + 보안 기초

### 13.1 보안 이슈 (긴급)

```
⚠️ 1. 테스트 계정 3개가 프로덕션 빌드에 포함됩니다
   - test@dugout.com / test1234
   - test2@dugout.com / test1234
   - admin@dugout.com / admin1234
   → APK/IPA 디컴파일로 누구나 확인 가능

   - admin@dugout.com / admin1234
   → 브라우저 개발자 도구에서 즉시 확인 가능

   → 환경변수 미설정 시 fallback으로 사용될 수 있음

### 13.2 아키텍처 현황

```
ℹ️ 1. 앱의 핵심 기능 중 커뮤니티/알림/메시지/어드민이 100% 목 데이터입니다
   → 새로고침하면 모든 변경사항 초기화
   → 실제 서비스 불가 상태

ℹ️ 2. 어드민 웹 전체가 100% 목 데이터입니다
   → Supabase 연동이 전혀 없음
   → 실제 관리 작업 불가

ℹ️ 3. Edge Functions가 0개입니다
   → 서버사이드 로직이 전혀 없음
   → 스팸 필터, 트렌딩 계산, 이미지 처리, 알림 발송 모두 미구현

ℹ️ 4. 프로덕션 준비도는 약 25% 수준입니다
   → UI는 90%+ 완성
   → 백엔드 연동이 핵심 작업
```
